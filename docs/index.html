¿Qué es Librería Químera?

Una plataforma web simple, desarrollada desde cero, donde los usuarios pueden:

  • Registrarse y gestionar su sesión
  • Subir libros en formato texto plano o escritos a mano
  • Consultar una biblioteca compartida de libros
  • Leer los libros desde el navegador
  • Dejar comentarios en los libros leídos
Se ha diseñado con una interfaz inspirada en lo medieval, y se ejecuta de forma contenida y automatizada para facilitar su uso y despliegue.

+Info

¿Para qué sirve?

Objetivo funcional del proyecto:

  • Crear una pequeña red de lectores y escritores online
  • Permitir compartir contenido de forma libre y segura
  • Servir como base para una futura plataforma más completa
  • Ser una práctica realista para trabajar con tecnologías modernas de desarrollo y sistemas
Además, como entorno de prueba, permite simular situaciones reales de despliegue, gestión de usuarios, base de datos y automatización.

+Info

Además, me daba la posibilidad de aprender por mi cuenta sobre contenedores, CI/CD y buenas prácticas modernas

¿Por qué este proyecto?

Entre la inspiración por mi gusto en la lectura y género medieval. Elegí este proyecto porque:

  • Quería hacer algo útil y visual, más allá de lo puramente técnico
  • Me permitía aplicar muchos conocimientos aprendidos en el ciclo: programación, bases de datos, sistemas, redes, automatización...
  • La idea de hacer una plataforma completa me motivó más que hacer una herramienta aislada

+Info

El flujo básico es:

  1. El usuario entra y se registra o inicia sesión
  2. Puede subir libros en texto o escribirlos manualmente
  3. La plataforma guarda el libro y lo muestra en la biblioteca
  4. Otros usuarios pueden leerlo y dejar comentarios

Arquitectura y funcionamiento

Estructura del sistema:

Frontend: HTML + CSS personalizado (estilo medieval) Backend: PHP 8.2 con lógica de negocio y conexión a base de datos Base de datos: MySQL 8, contenedorizada y gestionada desde Docker Docker Compose: Orquesta los servicios (backend, BBDD, red)

También destacar alguna frase o dato concreto que se quede grabado a fuego en la memoria de tu público e incluso embeber contenido externo que sorprenda: vídeos, fotos, audios... ¡Lo que tú quieras! ¿Necesitas más motivos para crear contenidos dinámicos? Aprendizajes personales

Este proyecto me ha permitido aplicar muchos conocimientos teóricos de forma práctica y comprobar que puedo llevar a cabo una aplicación funcional por mí mismo.

  • Diseñar un sistema desde cero, pensando tanto en el usuario como en el despliegue
  • Trabajar de forma ordenada, estructurando carpetas, código y estilos
  • Utilizar Docker Compose para simular un entorno profesional
  • Automatizar procesos reales con GitHub Actions
  • Integrar mis propios estilos visuales con funcionalidad moderna
  • Resolver problemas reales como errores de conexión, permisos, rutas y despliegue

+Info

Nuestro cerebro está preparado para consumir contenidos visuales. Algunos datos: el 90% de la información que procesamos nos llega a través de la vista y procesamos el contenido visual hasta 60.000 veces más rápido que el texto. Por eso la comunicación visual es más eficaz.

Lo que lees: la interactividad y la animación pueden hacer que el contenido más aburrido se convierta en algo divertido. En Genially utilizamos AI (Awesome Interactivity) en todos nuestros diseños, para que subas de nivel con interactividad y conviertas tu contenido en algo que aporta valor y engancha.

Captamos mejor el contenido visual. El contenido visual está asociado a mecanismos cognitivos y psicológicos. Las cosas entran por los ojos, la primera imagen es la que cuenta. Asociamos el contenido visual con emociones.

La interactividad y la animación pueden ser tus mejores aliadas a la hora de crear tablas, infografías o gráficos que ayuden a dar contexto a la información y simplificar los datos para traducirlos ante tu público. Somos seres visuales y nos resulta más sencillo 'leer' imágenes, que leer un texto escrito.
